Create and Expand Clusters with the cf-mgmt Tool

Not all cluster management and creation features are available through the Horizon console. The cf-mgmt tool can take direct input or leverage a bootstrap file to simplify cluster creation and expansion.

The following are examples on how to create and grow clusters using the cf-mgmt tool:

  • Create a small cluster with one core node and one DEA.
  • Add a DEA node to an existing cluster.
  • Add a service to an existing cluster.

Prerequisites

  • The Helion Development Platform must be installed.
  • The cf-mgmt tool must be downloaded and installed.
  • If creating clusters that support Keystone v3, you will need to modify the configuration file first.

Tips

  • To minimize the risk of errors, best practice is to create a small, starter cluster and then expand it, not create a huge cluster all at once.
  • For convenience, copy the code blocks into a text editor, edit them there, and then paste back into the cf-mgmt tool as necessary.
  • You may also want to refer to the full cf-mgmt tool command reference documentation.
