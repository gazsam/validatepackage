cf-mgmt Tool Example: Add a Service to an Existing Cluster

This example demonstrates how to add a service to a cluster that already exists.

Prerequisites

  • The host for the examples needs to be able to reach your HPE Helion OpenStack environment.
  • Download the RC configuration file.
  • Download the version of the cluster management (cf-mgmt) tool appropriate for your operating system.

Create a Bootstrap Configuration File

  1. Download the latest bootstrap.yml file and save it in a location where the cf-mgmt tool can access it.
  2. Rename bootstrap.yml to bootstrap_dea.yml
  3. Open and configure the bootstrap_dea.yml file to match your environment. Ensure the values for image names, image sizes and availability zones ("az") are valid for your current configuration. Here is an example: 
    constructor-image-name: HPE Helion Development Platform - Application Lifecycle Service Installer 1.2.0.282 
constructor-flavor: standard.xsmall 
seed-node-image-name: HPE Helion Development Platform - Application Lifecycle Service Seed Node 1.2.0.282 
core-flavor: standard.medium 
cluster-title: <My Cluster Title> 
cluster-prefix: <My Cluster Prefix>
services-on-core: false 
flavor: standard.medium
    Note: To add an existing Trove database, also include a value for the --database-instance-id parameter.
  4. Run the following command, inserting the correct values as appropriate.
    • This example will work for the WindowsÂ® operating system.
      cf-mgmt.exe ^
   --os-auth-url <auth-url> ^
   --os-username <username> ^
   --os-password <password> ^ 
   --os-tenant-id <id> ^
   --os-tenant-name "<tenant-name>" ^
   --os-region-name "<region-name>" ^
  add-service mysql ^
   --keypair-name <keypair name> ^
   --load bootstrap_service.yml
    • This example will work on Mac OSX or Linux: 
      source <path to openstackrc file>
cf-mgmt.exe \
  add-service mysql \
   --keypair-name <keypair name> \
   --load bootstrap_service.yml
