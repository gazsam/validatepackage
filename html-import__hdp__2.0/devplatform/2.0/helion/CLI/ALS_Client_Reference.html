Configuration Values

There are three ways to pass configuration values into ALS:

  • Direct entry through the cf-mgmt tool
  • Defined in an environment variable
  • Recorded in a configuration file

Configuration Files

ALS maintains separate configuration files, so configuration values must be included in the appropriate file.
Note: When creating clusters that support Keystone v3, you will need to modify the configuration file first.

One configuration file contains the values for the global variables. The global configuration file is located in the home directory and is named cf-mgmt.yml.

  • Global level commands create, delete, and add entire clusters.
  • Global level configurations manage tenants, users, versions, and other system information on single or multiple clusters.

The other configuration files, specified by the --load option, includes the values that should be passed as arguments to a specific command.

Debugging

Syntax: --debug [option]

This command provides debug information and can be very helpful when creating clusters.

The options are general, cpi, shell, kato, timing, all.

Example: 
cf-mgmt --debug all cc

Cluster Commands

These commands are available from the command line interface.

Use Syntax

cf-mgmt [global options] command [command options] [arguments...]

Command Options

Command Description
create-cluster, cc Creates a cluster
delete-cluster, dc Deletes a cluster
list-clusters, lc Lists clusters within a tenant.
show-cluster, sc Display detailed cluster information.
add-service, as Add services to an existing cluster.
add-role, ar Add node of the specified role to an existing cluster.
remove service, rs Remove services from an existing cluster.
remove role, rr Remove node of the specified role from an existing cluster.
update, up Update to the most recent version of cf-mgmt
help, h Displays a list of available commands or help for a command.
