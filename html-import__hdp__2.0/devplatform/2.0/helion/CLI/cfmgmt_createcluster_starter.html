cf-mgmt Tool Example: Create a Starter Cluster

The following is an example on how to create a small starter cluster with a core node and one DEA.

You can create any number of identical clusters using the same bootstrap file. However, each configuration requires its own file. For example, if you need five medium and five small clusters, you would need two bootstrap files; one to create five identical medium clusters, and one to create five small.

Prerequisites

  • The host for the examples needs to be able to reach your HPE Helion OpenStack environment.
  • Download the RC configuration file.
  • Download the version of the cluster management (cf-mgmt) tool appropriate for your operating system.
  1. Download the latest bootstrap.yml and save it in a location where the cf-mgmt tool can access it.
  2. Open and configure the bootstrap.yml file to match your environment. Here is an example:
    constructor-image-name: HPE Helion Development Platform - Application Lifecycle Service Installer 1.2.0.282 
constructor-flavor: standard.xsmall 
seed-node-image-name: HPE Helion Development Platform - Application Lifecycle Service Seed Node 1.2.0.282 
core-flavor: standard.medium 
cluster-title: <My Cluster Title> 
cluster-prefix: <My Cluster Prefix>
dea-count: 1 
dea-flavor: standard.large
az: <My Availability Zones>
    Note: Please ensure the values for image names, image sizes and availability zones ("az") are valid for your current configuration.
  3. Run the following command, inserting the correct values as appropriate.
    • This example will work for the WindowsÂ® operating system.
       cf-mgmt.exe ^
        --os-auth-url <OpenStack authentication URL> ^
        --os-username <username> ^
        --os-password <password> ^
        --os-tenant-id <id> ^
        --os-tenant-name "<tenant-name>" ^
        --os-region-name "<region-name>" ^
 create-cluster ^
        --keypair-name <keypair name> ^
        --admin-email <admin email> ^
        --admin-password <admin password> ^
        --load bootstrap.yml
    • This example will work on Mac OSX or Linux: 
      source <path to openstackrc file>
cf-mgmt create-cluster \
   --keypair-name <name of keypair> \
   --admin-email <your login> \
   --admin-password <your password> \
   --load bootstrap.yml
Note: If you are using self-signed certificates, add the --skip-ssl-validation option before the create-cluster command.
