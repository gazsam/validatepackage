HPE Helion 2.0 Development Platform: Testing

Each of the tech stacks and frameworks has conventions for supporting unit testing.

The HPE Helion Code Engine looks for specific files in the root directory of the project. The specific file found will determine how the project is built and tested.

Note: Only one of these files is expected to exist in the project root directory. The behavior is undefined in the presence of multiple files.

Java Projects

pom.xml
Build: mvn -Dmaven.test.skip=true package
Test: mvn test
gradle.build
Build: gradle build -x test
Test: gradle test
gradlew
Build:
Test: gradlew test

Python projects

tox.ini
Build: python setup.py install
Test: tox
setup.py
Build: python setup.py install
Test: python setup.py test

NodeJS Projects

Build: npm install
Test: npm test
