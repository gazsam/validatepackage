Monitoring the messaging service from the Operations Console

This topic describes monitoring information available through the Operations Console.

The monitoring console can generate the following alarms:
Alarm name Description Likely Cause Mitigation tasks to perform
HTTP Status component=cue-api API is unresponsive Restart Apache
Process Check process_name=cue-worker Supervisor stopped Use supervisor to start all workers
Process Check process_name=cue-monitor cue-monitor process crashed Restart cue-monitor service
Process Check process_name=mysqld mysql process crashed Restart mysql service
Process Check process_name=org.apache.zookeeper.server zookeeper process crashed Restart zookeeper service
Note: When looking at the Ops Console, alarms for MySQL and Zookeeper appear in the No Service Defined panel, not the Messaging as a Service panel. This is a known issue.
